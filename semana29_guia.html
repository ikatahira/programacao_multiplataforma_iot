✅ Resoluções das Atividades

Atividade 1: Resolução Modelo

Exemplos de Apps para Análise:

  • Instagram: React Native - Performance, UX fluida
  • Alibaba: Flutter - Multiplataforma consistente
  • Tesla: React Native - Interface moderna

Estrutura do Relatório:

1. Introdução (explicar tecnologia escolhida)

2. Análise de cada app (funcionalidades, pontos fortes/fracos)

3. Comparação entre os apps

4. Conclusões e aprendizados

💡 Dica Pedagógica: Incentive os alunos a testarem os apps em diferentes dispositivos e anotarem diferenças de comportamento.

Atividade 2: Resolução Modelo

Exemplo de Código Base:

// Estrutura básica do projeto
src/
  components/
    Header.js
    Footer.js
  screens/
    HomeScreen.js
    AboutScreen.js
  navigation/
    AppNavigator.js
  App.js

O código deve incluir: componentes funcionais, hooks, navegação básica e estilização.

💡 Dica Pedagógica: Forneça um template inicial para alunos com dificuldades, mas incentive todos a personalizarem completamente.

Atividade 3: Resolução Modelo

Pontos para Discussão em Grupo:

  • Vantagens: código único, redução de custos, time to market
  • Desvantagens: performance, acesso a APIs nativas, tamanho do app
  • Quando usar: MVPs, apps simples, projetos com budget limitado
  • Quando evitar: apps com requisitos nativos específicos, jogos pesados
💡 Dica Pedagógica: Modere a discussão garantindo que todos participem. Use técnicas como "round robin" para dar voz a todos.
