📦 Reunir Materiais

Antes de começar, separe todos os componentes necessários:

Lista de Componentes:

  • 1× ESP32 (qualquer variante: DevKit V1, WROOM, WROVER…)
  • 8× LED Vermelho (DLED1 a DLED8)
  • 8× Resistor 330 Ω (RR1 a RR8) — faixas laranja-laranja-marrom
  • Protoboard (830 pontos recomendado)
  • Jumpers macho-macho (pelo menos 12)
  • Cabo USB (Micro-USB ou USB-C, conforme modelo do ESP32)
  • Computador com Arduino IDE instalado + suporte ao ESP32

⚡ Diferença ESP32 × Arduino Uno

  • O ESP32 opera em 3,3 V (não 5 V!). Os pinos GPIO fornecem 3,3 V — suficiente para LEDs comuns com resistor de 330 Ω.
  • Os pinos GPIO do ESP32 suportam até 40 mA por pino, mas recomenda-se ≤ 12 mA por segurança.
  • O ESP32 não vem com suporte na Arduino IDE por padrão — é preciso instalar o pacote de placas.
💡 Dica: Não tem os materiais físicos? Use o simulador online Wokwi — ele suporta ESP32!
Acesse: wokwi.com (mais completo que Tinkercad para ESP32)
⚠️ Atenção — Pinos reservados do ESP32:
Evite usar GPIO 0, 1, 3, 6–11 para LEDs. Eles têm funções especiais (boot, UART, flash).
Neste projeto usamos: GPIO 2 GPIO 4 GPIO 5 GPIO 18 GPIO 19 GPIO 21 GPIO 22 GPIO 23

🔢 Entendendo Números Binários

Antes de montar, vamos entender o que o circuito vai fazer!

🧠 O que é binário?

No sistema decimal usamos 10 dígitos (0 a 9). No sistema binário usamos apenas 2 dígitos: 0 e 1. Um LED apagado = bit 0; LED aceso = bit 1.

    ⚙️ Configurando o Arduino IDE para o ESP32

    ⚡ Passo extra em relação ao Arduino Uno

    O ESP32 não é reconhecido automaticamente pelo Arduino IDE. Precisamos instalar o pacote de placas ESP32 da Espressif.

    1. Adicionar a URL do ESP32

    Abra o Arduino IDE → Arquivo → Preferências → no campo "URLs adicionais para gerenciadores de placas", cole:

    https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

    2. Instalar as placas ESP32

    Ferramentas → Placa → Gerenciador de Placas…
    Pesquise "esp32" → instale o pacote "esp32 by Espressif Systems".

    3. Selecionar a placa correta

    Ferramentas → Placa → ESP32 Arduino → ESP32 Dev Module
    (ou o modelo exato que você tem: ESP32-WROOM-32, NodeMCU-32S…)

    4. Configurar o baudrate do Monitor Serial

    ⚠️ Importante: O ESP32 usa 115200 bps por padrão (o Arduino Uno usava 9600).
    No Monitor Serial, mude o baudrate para 115200 — senão você verá caracteres embaralhados!

    5. Verificar porta COM

    Ferramentas → Porta → selecione a porta onde o ESP32 está conectado.

    💡 Dica: Se a porta não aparecer, instale o driver do chip USB do ESP32:
    • CP2102 (Silicon Labs): silabs.com
    • CH340 (clone): wch.cn
  • URL do ESP32 adicionada nas preferências
  • Pacote "esp32 by Espressif" instalado
  • Placa "ESP32 Dev Module" selecionada
  • Porta COM correta selecionada

    ⚡ Por que esses GPIOs específicos?

    • GPIO 0 → controla modo boot. Evitar como saída.
    • GPIO 1 e 3 → TX/RX da UART0 (Monitor Serial). Não usar para LEDs.
    • GPIO 6–11 → conectados à memória flash interna. Nunca usar.
    • GPIO 2, 4, 5, 18, 19, 21, 22, 23 → GPIOs seguros para saída digital. ✅

    🔧 Montagem — Parte 1: Posicionando os LEDs

    Vamos montar os 8 LEDs na protoboard de forma organizada, um ao lado do outro.

    Disposição dos LEDs na Protoboard

    Protoboard — linha A (catodos dos LEDs): Col: 3 6 9 12 15 18 21 24 | | | | | | | | A- [LED1][LED2][LED3][LED4][LED5][LED6][LED7][LED8] ↑ ↑ ↑ ↑ ↑ ↑ ↑ ↑ Ânodo (+) Cátodo (-) = perna mais curta → GND Cada LED ocupa 2 linhas na protoboard. Perna longa (+) → fileira A (linha de cima) Perna curta (-) → fileira B (linha de baixo → vai para GND via resistor)

    📋 Instruções passo a passo:

    1. Posicione o ESP32 na protoboard deixando espaço à direita
    2. Insira o LED1 na coluna 3 (perna longa na fileira A, perna curta na B)
    3. Repita para os outros 7 LEDs nas colunas 6, 9, 12, 15, 18, 21, 24
    4. Todos os LEDs devem ficar na mesma orientação (ânodo voltado para cima)
    ⚠️ Polaridade: O LED tem polaridade! Perna longa = ânodo (+). Se inverter, o LED não acende — e não danifica com resistor, mas não funciona.
  • 8 LEDs inseridos na protoboard
  • Todos na mesma orientação (perna longa = ânodo voltado igual)
  • Há espaço entre eles para inserir os resistores

    🔧 Montagem — Parte 2: Resistores de 330 Ω

    Por que 330 Ω com o ESP32?

    ESP32 GPIO: 3,3 V | LED vermelho: ~1,8 V | I máx: ~10 mA Cálculo do resistor: R = (Vcc - Vled) / I R = (3,3 V - 1,8 V) / 0,010 A R = 1,5 V / 0,010 A R = 150 Ω (mínimo) Usamos 330 Ω → corrente ~4,5 mA → seguro e ainda bem visível ✅ ⚠️ Com Arduino Uno (5V): R = (5 - 1,8) / 0,010 = 320 Ω → 330 Ω também ok Com ESP32 (3,3V): 330 Ω → corrente menor → LED um pouco menos brilhante, mas funcional

    📋 Inserindo os resistores:

    1. Cada resistor vai em série com o cátodo do LED
    2. Uma perna do resistor conecta ao cátodo do LED (fileira B)
    3. A outra perna do resistor conecta ao trilho negativo (GND) da protoboard
    4. Repetir para os 8 LEDs
    💡 Código de cores — 330 Ω:
    Laranja – Laranja – Marrom – Dourado (tolerância 5%)
  • 8 resistores de 330 Ω inseridos, cada um em série com um LED
  • Uma perna de cada resistor ligada ao trilho GND da protoboard

    🔧 Montagem — Parte 3: Conectar GND

    ⚡ GND no ESP32

    O ESP32 DevKit tem múltiplos pinos GND na placa. Use o mais conveniente (geralmente o mais próximo dos LEDs).

    Conexão do GND:

    1. Pegue um jumper e conecte um pino GND do ESP32 ao trilho negativo (−) da protoboard
    2. Verifique se os resistores já estão conectados a esse trilho negativo
    3. Confirme com o multímetro (ou visualmente) que o trilho negativo está contínuo
    ⚠️ NÃO conecte o trilho positivo da protoboard ao 3,3 V ou 5 V ainda.
    Os LEDs são controlados pelos GPIOs — o trilho positivo não é necessário neste projeto.
  • Jumper do GND do ESP32 conectado ao trilho negativo da protoboard
  • Todos os resistores já conectados ao trilho negativo

    💻 Código para o ESP32

    ⚡ O que mudou em relação ao código do Arduino Uno?

    • Pinos definidos em um array — mais organizado e fácil de alterar
    • Serial.begin(115200) ao invés de 9600
    • Intervalo válido garantido por código (if num < 0 / > 255)
    • Mensagens descritivas no Monitor Serial para facilitar o debug

    Código completo para ESP32:

    📄 projeto-leds-esp32.ino
    // Array com os 8 pinos GPIO do ESP32 // Ordem: MSB (bit 7) ao LSB (bit 0) const int pinos[8] = {2, 4, 5, 18, 19, 21, 22, 23}; int num = 0; void setup() { Serial.begin(115200); for (int i = 0; i < 8; i++) { pinMode(pinos[i], OUTPUT); digitalWrite(pinos[i], LOW); } Serial.println("Conversor Decimal → Binario | ESP32"); Serial.println("Digite um numero de 0 a 255:"); } void loop() { if (Serial.available() > 0) { num = Serial.parseInt(); // Limpa buffer while (Serial.available()) { Serial.read(); } // Garante intervalo valido if (num < 0) num = 0; if (num > 255) num = 255; Serial.print("Recebido: "); Serial.println(num); Serial.print("Binario: "); // Usa manipulacao de bits (mais eficiente) for (int i = 0; i < 8; i++) { int bitValue = bitRead(num, 7 - i); digitalWrite(pinos[i], bitValue); Serial.print(bitValue); } Serial.println(); Serial.println("Digite outro numero (0–255):"); } }
  • Código copiado para o Arduino IDE

  • Código verificado (Ctrl+R) sem erros

    🚀 Carregar o Código e Testar

    1. Carregar o código

    Clique em Upload (→) ou pressione Ctrl+U.

    ⚡ Dica ESP32 — Modo de Boot

    Se o upload travar em "Connecting...", pressione e segure o botão BOOT/IO0 do ESP32 por 1–2 segundos enquanto o upload começa. Alguns modelos necessitam disso.

    2. Abrir o Monitor Serial

    Ferramentas → Monitor Serial (ou Ctrl+Shift+M)

    ⚠️ Fundamental: Mude o baudrate no Monitor Serial para 115200. Se deixar em 9600, você verá lixo na tela!

    3. Testar

    No Monitor Serial, digite um número entre 0 e 255 e pressione Enter. Os LEDs devem acender mostrando o valor em binário!

    Monitor Serial — saída esperada (115200 bps): ============================== Conversor Decimal → Binário ESP32 + 8 LEDs ============================== Digite um número de 0 a 255: > 42 Recebido: 42 → Binário: 00101010 Digite outro número (0–255): > 255 Recebido: 255 → Binário: 11111111 Digite outro número (0–255):

    🟡 Desafio 1 — Contador automático:

    Faça o ESP32 contar de 0 a 255 automaticamente:

    void loop() { for(int i = 0; i <= 255; i++) { mostrarBinario(i); // extraia o algoritmo para uma função delay(500); } }

    🟢 Desafio 2 — Potência de 2 (bit caminhando):

    for(int i = 0; i < 8; i++) { int valor = 1 << i; // 2^i via deslocamento de bits mostrarBinario(valor); delay(300); }
    💡 Lembre-se: Verificar (Ctrl+R) → Carregar (Ctrl+U) → Observar resultado nos LEDs!
