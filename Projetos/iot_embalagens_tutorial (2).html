PROJETO 01 · BIODEGRADÁVEL
Monitor de Saúde da "Embalagem Viva" (Micélio)
Garante que o substituto biodegradável do isopor cresça com as condições ideais
+120 XP
▼
▸ CONTEXTO DO PROBLEMA

O isopor (poliestireno expandido) é um dos maiores vilões das embalagens de e-commerce — leva centenas de anos para se decompor e raramente é reciclado. A solução? Embalagens de micélio (fungos + substrato agrícola) que são 100% compostáveis. Mas para crescer corretamente, o micélio precisa de temperatura e umidade específicas. Sem monitoramento, o material cresce mal e vira desperdício de matéria-prima.

1
Entenda o Hardware
Você vai usar 3 componentes principais:
COMPONENTEPINO ESP32FUNÇÃO
ESP32—Microcontrolador principal
DHT22GPIO 4Temperatura e Umidade do ar
Sensor Umidade SoloGPIO 34 (ADC)Umidade do substrato de fungos
2
Monte as Conexões (Pinout)
▸ DIAGRAMA DE PINOS
DHT22 DATA──────→GPIO 4
DHT22 VCC──────→3.3V
DHT22 GND──────→GND
Sensor Solo AO──────→GPIO 34 (entrada analógica)
Sensor Solo VCC──────→3.3V
Sensor Solo GND──────→GND
⚠️
O GPIO 34 do ESP32 é somente entrada (input-only) e não tem resistor pull-up interno. Use-o apenas para leitura analógica.
3
Carregue o Código no ESP32
💡
Cole o código abaixo no Arduino IDE. Verifique se a biblioteca DHT sensor library (Adafruit) está instalada antes de compilar.
C++ / ARDUINO
#include "DHT.h"

// Definição dos pinos
#define DHTPIN    4
#define SOLOPIN  34
#define DHTTYPE  DHT22

DHT dht(DHTPIN, DHTTYPE);

// Faixas ideais para crescimento do micélio
const float UMID_AR_MIN  = 80.0;
const float UMID_AR_MAX  = 95.0;
const float TEMP_MIN     = 20.0;
const float TEMP_MAX     = 26.0;

void setup() {
  Serial.begin(115200);
  dht.begin();
  Serial.println("Monitor de Micélio iniciado...");
}

void loop() {
  float umidadeAr   = dht.readHumidity();
  float temperatura = dht.readTemperature();
  int   umidadeSolo = analogRead(SOLOPIN); // 0-4095

  // Verificar se a leitura é válida
  if (isnan(umidadeAr) || isnan(temperatura)) {
    Serial.println("ERRO: Falha na leitura do DHT22!");
    delay(2000);
    return;
  }

  Serial.println("=== MONITOR MICÉLIO ===");
  Serial.printf("Temp:     %.1f°C  [Ideal: 20-26°C]\n", temperatura);
  Serial.printf("Umid. Ar: %.1f%%   [Ideal: 80-95%%]\n", umidadeAr);
  Serial.printf("Solo:     %d/4095  [Úmido se > 2000]\n", umidadeSolo);

  // Alertas de fora do range ideal
  if (temperatura < TEMP_MIN || temperatura > TEMP_MAX)
    Serial.println("⚠ ALERTA: Temperatura fora do ideal! Ajuste ventilação.");
  if (umidadeAr < UMID_AR_MIN)
    Serial.println("⚠ ALERTA: Ar muito seco! Borrife água no substrato.");
  if (umidadeSolo < 1000)
    Serial.println("⚠ ALERTA: Solo seco! Umidifique o substrato.");

  Serial.println();
  delay(2000);
}
4
Interprete os Resultados
Abra o Monitor Serial (115200 baud). Você verá os dados em tempo real. O sensor de solo retorna valores de 0 a 4095: quanto maior o valor, mais úmido o substrato. Valores abaixo de 1000 indicam solo seco — hora de umidificar. Temperatura ideal é entre 20–26°C.
~100%
de substituição do isopor é possível com micélio bem cultivado — embalagem compostável em 45 dias.
🎤
No Pitch: "Nossa tecnologia garante a viabilidade de embalagens 100% compostáveis, eliminando o plástico bolha. Com o sensor monitorando em tempo real, reduzimos a taxa de falha no cultivo do micélio de ~40% para menos de 5%."

PROJETO 02 · ANTI-DESPERDÍCIO
Otimizador de Enchimento (Anti-Espaço Vazio)
Alerta quando a caixa escolhida é grande demais para o produto — elimina o uso de enchimento
+100 XP
▼
▸ CONTEXTO DO PROBLEMA

Um estudo mostrou que até 40% do volume de uma caixa de e-commerce é ar. Esse espaço vazio é preenchido com plástico bolha, papel amassado ou isopor — material que vai direto para o lixo. A solução: um sensor que mede o espaço livre dentro da caixa antes de fechar e alerta se a caixa está grande demais para o produto.

1
Hardware Necessário
COMPONENTEPINO ESP32FUNÇÃO
HC-SR04 (Ultrassônico)TRIG: GPIO 5 · ECHO: GPIO 18Mede distância até o produto
2
Monte as Conexões
▸ DIAGRAMA DE PINOS — HC-SR04
VCC──→5V (use pino VIN ou fonte externa 5V)
TRIG──→GPIO 5
ECHO──→GPIO 18 via divisor de tensão (1kΩ + 2kΩ)
GND──→GND
⚠️
O HC-SR04 opera a 5V e o pino ECHO retorna 5V — conectar direto no GPIO do ESP32 (3.3V) pode danificá-lo. Use um divisor de tensão: R1=1kΩ entre ECHO e GPIO, R2=2kΩ entre GPIO e GND.
3
Código do Projeto
💡
Posicione o sensor HC-SR04 na tampa da caixa, apontando para baixo. A distância lida = espaço vazio acima do produto. Defina um limite máximo aceitável (ex: 15 cm).
C++ / ARDUINO
// Projeto 2 — Otimizador de Enchimento
const int TRIG = 5;
const int ECHO = 18;
const int LIMITE_VAZIO_CM = 15; // Máximo de espaço aceitável

void setup() {
  Serial.begin(115200);
  pinMode(TRIG, OUTPUT);
  pinMode(ECHO, INPUT);
  Serial.println("Otimizador de Enchimento iniciado.");
}

long medirDistancia() {
  digitalWrite(TRIG, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG, LOW);
  long duracao = pulseIn(ECHO, HIGH, 30000); // timeout 30ms
  return duracao * 0.034 / 2; // converte para cm
}

void loop() {
  long dist = medirDistancia();

  Serial.printf("Espaço vazio: %ld cm\n", dist);

  if (dist > LIMITE_VAZIO_CM) {
    Serial.println("🚨 ALERTA: Caixa GRANDE DEMAIS!");
    Serial.println("   → Use uma embalagem menor para este produto.");
    Serial.println("   → Isso evita desperdício de enchimento.");
  } else {
    Serial.println("✅ OK: Tamanho de caixa adequado para o produto.");
  }
  delay(1000);
}
4
Teste e Calibre
Coloque objetos de tamanhos diferentes na caixa e observe os alertas no Monitor Serial. Ajuste a constante LIMITE_VAZIO_CM de acordo com o tamanho padrão das caixas do seu projeto. Objetos com mais de 15 cm de folga devem gerar alerta.
-40%
de redução no uso de materiais de preenchimento ao forçar a escolha da caixa correta para cada produto.
🎤
No Pitch: "Reduzimos em até 40% o uso de materiais de preenchimento ao forçar a escolha da caixa correta. Isso elimina plástico bolha e papel amassado em cada envio."

PROJETO 03 · LOGÍSTICA CIRCULAR
Smart Tag RFID — Gestão de Reuso de Embalagens
Rastreia quantas vezes cada caixa retornável foi usada e prova sua viabilidade ambiental
+130 XP
▼
▸ CONTEXTO DO PROBLEMA

Cada caixa de papelão é usada em média 1,2 vezes antes de virar resíduo. Caixas plásticas retornáveis podem durar 50+ reusos, mas precisam de rastreamento para provar sua viabilidade ambiental e financeira. Com RFID, cada caixa tem uma identidade digital e seu ciclo de vida é monitorado automaticamente.

1
Hardware Necessário
COMPONENTEPINO ESP32FUNÇÃO
Módulo RFID RC522SPI (ver diagrama)Lê as tags nas caixas
Tags RFID (cartão/chaveiro)—Identificador único de cada caixa
2
Diagrama SPI do RC522
▸ CONEXÕES SPI — RFID RC522
SDA (SS)──→GPIO 5
SCK──→GPIO 18
MOSI──→GPIO 23
MISO──→GPIO 19
RST──→GPIO 22
VCC──→3.3V
GND──→GND
ℹ️
O RC522 opera a 3.3V — não use 5V, pois o módulo será danificado. Cole ou fixe a tag RFID na lateral de cada caixa retornável.
3
Código do Projeto
C++ / ARDUINO
#include <SPI.h>
#include <MFRC522.h>

MFRC522 rfid(5, 22); // SDA=5, RST=22

// Contador de usos por ID (simulado em RAM)
// Em produção: usar EEPROM ou servidor MQTT
int contadorReuso = 0;
String ultimoID = "";

void setup() {
  Serial.begin(115200);
  SPI.begin();
  rfid.PCD_Init();
  Serial.println("Sistema RFID de Rastreio de Embalagens ativo.");
  Serial.println("Aproxime a embalagem do leitor...");
}

void loop() {
  if (!rfid.PICC_IsNewCardPresent() || !rfid.PICC_ReadCardSerial()) {
    delay(200);
    return;
  }

  // Montar string do UID
  String idAtual = "";
  for (byte i = 0; i < rfid.uid.size; i++) {
    idAtual += String(rfid.uid.uidByte[i], HEX);
    if (i < rfid.uid.size - 1) idAtual += ":";
  }
  idAtual.toUpperCase();

  if (idAtual != ultimoID) {
    ultimoID = idAtual;
    contadorReuso++;
  }

  Serial.println("\n=== EMBALAGEM IDENTIFICADA ===");
  Serial.printf("ID da Caixa:     %s\n", idAtual.c_str());
  Serial.printf("Ciclos de reuso: %d\n", contadorReuso);

  if (contadorReuso >= 50)
    Serial.println("✅ PONTO DE EQUILÍBRIO: Embalagem já se pagou ambientalmente!");
  else
    Serial.printf("   → Faltam %d reusos para atingir ponto de equilíbrio.\n", 50 - contadorReuso);

  rfid.PICC_HaltA();
  rfid.PCD_StopCrypto1();
  delay(1500);
}
4
Evolução: Persistir os Dados
O código acima conta reusos na RAM (perde ao reiniciar). Para um protótipo mais completo, salve o contador na EEPROM do ESP32 ou envie via Wi-Fi para um servidor. Isso prova a rastreabilidade ao longo do tempo. Cada tag RFID tem um ID único gravado de fábrica — você não precisa programar as tags, só lê-las.
50x
é o número de reusos necessário para que a caixa plástica retornável se pague ambientalmente em relação ao papelão descartável.
🎤
No Pitch: "Transformamos a embalagem de um resíduo em um ativo rastreável que volta para a empresa. Com nossa Smart Tag, sabemos exatamente quantas vezes cada caixa foi usada e quando ela se pagou ambientalmente."

PROJETO 04 · FIM DA FITA
Lacre Digital Reutilizável
Substitui metros de fita adesiva plástica por uma trava mecânica monitorada e reutilizável
+140 XP
▼
▸ CONTEXTO DO PROBLEMA

Cada caixa de e-commerce usa em média 1,5 a 3 metros de fita adesiva plástica que não é reciclável. Esse plástico vai direto para aterros. A solução é uma trava mecânica inteligente: a caixa se fecha e trava por um micro servo controlado pelo ESP32, e só abre com um sinal digital — eliminando a fita completamente.

1
Hardware Necessário
COMPONENTEPINO ESP32FUNÇÃO
Micro Servo 9g (SG90)GPIO 13 (PWM)Aciona a trava mecânica
Reed Switch (sensor magnético)GPIO 14Detecta se a caixa está fechada
Ímã pequeno—Fixado na aba da caixa
2
Conexões
▸ DIAGRAMA DE PINOS
Servo SINAL──→GPIO 13
Servo VCC──→5V (pino VIN do ESP32)
Servo GND──→GND
Reed Switch──→GPIO 14 (INPUT_PULLUP)
Reed Switch (outro terminal)──→GND
💡
O Reed Switch é um interruptor magnético: fecha o circuito quando um ímã se aproxima. Fixe-o na borda da caixa e o ímã na aba. Quando a caixa fecha, o ímã aproxima do sensor e o GPIO vai para LOW.
3
Código do Projeto
C++ / ARDUINO
#include <ESP32Servo.h>

Servo trava;
const int SERVO_PIN  = 13;
const int REED_PIN   = 14;

const int POS_TRAVADO = 90;  // graus — trava acionada
const int POS_ABERTO  = 0;   // graus — trava recolhida

bool estadoAnterior = false;

void setup() {
  Serial.begin(115200);
  trava.attach(SERVO_PIN);
  pinMode(REED_PIN, INPUT_PULLUP);
  trava.write(POS_ABERTO);
  Serial.println("Lacre Digital iniciado. Caixa aberta.");
}

void loop() {
  bool caixaFechada = (digitalRead(REED_PIN) == LOW); // ímã próximo = LOW

  if (caixaFechada && !estadoAnterior) {
    // Acabou de fechar → ativa a trava
    trava.write(POS_TRAVADO);
    Serial.println("🔒 CAIXA FECHADA — Trava acionada. Fita desnecessária!");
    estadoAnterior = true;
  }
  else if (!caixaFechada && estadoAnterior) {
    // Caixa aberta (via comando/app) → libera trava
    trava.write(POS_ABERTO);
    Serial.println("🔓 CAIXA ABERTA — Trava liberada.");
    estadoAnterior = false;
  }

  delay(200);
}
4
Monte a Trava Física
Use a impressora 3D ou corte a laser da Sala Maker para criar a peça de trava que o servo vai mover. O servo gira 90°, encaixando um "dente" em um receptor na aba da caixa. Quando o cliente recebe o pacote, ele aciona a abertura via botão ou app — sem rasgar, sem fita.
1.5m
de fita adesiva plástica eliminada por envio. Em 1.000 pedidos/mês = 1,5 km de plástico evitado.
🎤
No Pitch: "Eliminamos 1,5 metros de fita adesiva de uso único por pacote. Nossa trava mecânica facilita a abertura e aumenta a chance de reuso da embalagem em 10x."

PROJETO 05 · LOGÍSTICA
Medidor de Cubagem Automático
Mede as 3 dimensões do produto para escolher sempre a menor embalagem possível
+150 XP
▼
▸ CONTEXTO DO PROBLEMA

Se medirmos o produto com precisão antes do empacotamento, podemos escolher automaticamente a menor caixa que o comporta. Isso elimina o espaço vazio e, consequentemente, elimina o plástico bolha. A medição 3D automática substitui o processo manual (impreciso) de escolha de caixas pelos operadores.

1
Hardware Necessário
COMPONENTEPINOS ESP32EIXO
HC-SR04 #1TRIG:5 / ECHO:18Comprimento (X)
HC-SR04 #2TRIG:16 / ECHO:17Largura (Y)
HC-SR04 #3TRIG:4 / ECHO:2Altura (Z)
💡
Use MDF ou madeira do corte a laser para construir uma "gaiola" cúbica com 50x50x50cm, com um sensor em cada face. O produto é colocado no centro e as 3 dimensões são lidas simultaneamente.
2
Código do Projeto
C++ / ARDUINO
// Projeto 5 — Medidor de Cubagem 3D
const int SUPORTE = 50; // cm — tamanho interno da gaiola

// Pinos: {TRIG, ECHO}
int sensores[3][2] = {{5,18}, {16,17}, {4,2}};
String eixos[3] = {"Comprimento", "Largura", "Altura"};

long lerDistancia(int trig, int echo) {
  digitalWrite(trig, LOW); delayMicroseconds(2);
  digitalWrite(trig, HIGH); delayMicroseconds(10);
  digitalWrite(trig, LOW);
  return pulseIn(echo, HIGH, 30000) * 0.034 / 2;
}

void setup() {
  Serial.begin(115200);
  for (int i = 0; i < 3; i++) {
    pinMode(sensores[i][0], OUTPUT);
    pinMode(sensores[i][1], INPUT);
  }
  Serial.println("Medidor de Cubagem 3D ativo. Posicione o produto.");
}

void loop() {
  delay(1000);
  Serial.println("\n=== MEDIÇÃO DO PRODUTO ===");

  long dims[3];
  long volume = 1;

  for (int i = 0; i < 3; i++) {
    long dist = lerDistancia(sensores[i][0], sensores[i][1]);
    dims[i] = SUPORTE - dist; // dimensão real = suporte - espaço vazio
    Serial.printf("%s: %ld cm\n", eixos[i].c_str(), dims[i]);
    volume *= dims[i];
  }

  Serial.printf("Volume: %ld cm³\n", volume);

  // Sugestão de tamanho de caixa
  if (volume < 1000)       Serial.println("→ Sugestão: Caixa PP (menor)");
  else if (volume < 8000)  Serial.println("→ Sugestão: Caixa P");
  else if (volume < 27000) Serial.println("→ Sugestão: Caixa M");
  else                     Serial.println("→ Sugestão: Caixa G");
}
3
Construa a Gaiola de Medição
Este é o projeto mais desafiador em termos de hardware físico. Use o corte a laser para criar peças de MDF que formam uma estrutura de 50×50×50 cm. Os três sensores são posicionados em faces perpendiculares. O produto é colocado no canto da gaiola e as leituras são feitas simultaneamente.

de papelão desperdiçado com embalagens superestimadas, ao usar a caixa certa para cada produto.

No Pitch: "Se medirmos o produto com precisão de milímetros, eliminamos a necessidade de caixas grandes e, consequentemente, eliminamos o plástico bolha. Este projeto ataca a raiz do problema."

PROJETO 06 · GAMIFICAÇÃO
Beacon de Devolução Incentivada (BLE)
Detecta via Bluetooth quando o cliente trouxe a embalagem para o ponto de coleta e libera recompensas
+110 XP
▼
▸ CONTEXTO DO PROBLEMA

Mesmo com embalagens retornáveis, o maior desafio é fazer o cliente devolvê-las. A solução é a gamificação: quando o smartphone do cliente chega perto do ponto de coleta, o ESP32 Beacon BLE detecta a presença e automaticamente libera pontos, créditos ou descontos — sem nenhum app especial necessário.

1
Hardware Necessário
Este projeto usa apenas o ESP32 — sem sensores externos. O rádio Bluetooth Low Energy (BLE) já vem integrado no chip. Você só precisa de: ESP32 + cabo USB para programar + fonte de energia (bateria 9V ou powerbank).
2
Código do Beacon BLE
C++ / ARDUINO
#include <BLEDevice.h>
#include <BLEUtils.h>
#include <BLEServer.h>
#include <BLEAdvertisedDevice.h>

// Nome que aparecerá nos dispositivos próximos
#define NOME_BEACON "PONTO_COLETA_ECO_01"

void setup() {
  Serial.begin(115200);

  // Inicializar BLE
  BLEDevice::init(NOME_BEACON);

  // Criar servidor BLE
  BLEServer *pServer = BLEDevice::createServer();

  // Configurar advertising
  BLEAdvertising *pAdv = BLEDevice::getAdvertising();
  pAdv->setScanResponse(true);
  pAdv->setMinPreferred(0x06); // iPhone compatibility
  pAdv->setMinPreferred(0x12);
  BLEDevice::startAdvertising();

  Serial.println("Beacon BLE ativo!");
  Serial.printf("Emitindo sinal como: %s\n", NOME_BEACON);
  Serial.println("Qualquer app BLE Scanner detectará este ponto.");
}

void loop() {
  // O beacon emite continuamente — sem necessidade de código no loop
  delay(1000);
}
3
Teste com App BLE Scanner
Instale o app "nRF Connect" ou "BLE Scanner" no celular. Quando estiver próximo do ESP32, o beacon aparecerá na lista com o nome "PONTO_COLETA_ECO_01". Para o pitch, isso demonstra como o sistema detectaria automaticamente a chegada do cliente com a embalagem. RSSI mais próximo de 0 = mais perto do beacon.
ℹ️
Para um sistema completo, o app do cliente consultaria um servidor ao detectar o beacon e liberaria os pontos/créditos automaticamente. No protótipo, basta mostrar a detecção do sinal BLE.

de aumento na taxa de devolução de embalagens reportado por programas de gamificação com recompensas imediatas.

PROJETO 07 · EMBALAGEM LEVE
Monitor de Integridade (Acelerômetro MPU6050)
Prova que embalagens mais finas protegem o produto — se o transporte for cuidadoso
+120 XP
▼
▸ CONTEXTO DO PROBLEMA

Embalagens mais grossas existem para proteger contra choques e quedas. Mas se o transporte for cuidadoso, podemos usar caixas com 20% menos papelão. O MPU6050 registra cada impacto durante a entrega, criando um "relatório de tratamento" da carga. Isso permite negociar com transportadoras e justificar o uso de embalagens mais leves.

1
Hardware Necessário
COMPONENTEPINO ESP32PROTOCOLO
MPU6050 (acelerômetro/giroscópio)SDA: GPIO 21 · SCL: GPIO 22I2C
2
Conexões I2C
▸ DIAGRAMA I2C — MPU6050

AD0──→GND (define endereço I2C como 0x68)
3
Código do Projeto
C++ / ARDUINO
#include <Adafruit_MPU6050.h>
#include <Adafruit_Sensor.h>
#include <Wire.h>

Adafruit_MPU6050 mpu;

const float LIMIAR_IMPACTO = 15.0; // m/s² — ajuste conforme necessário
int contagemImpactos = 0;

void setup() {
  Serial.begin(115200);

  if (!mpu.begin()) {
    Serial.println("MPU6050 não encontrado! Verifique conexões.");
    while (1) delay(10);
  }

  mpu.setAccelerometerRange(MPU6050_RANGE_8_G);
  mpu.setFilterBandwidth(MPU6050_BAND_21_HZ);

  Serial.println("Monitor de Integridade ativo. Aguardando transportes...");
}

void loop() {
  sensors_event_t a, g, temp;
  mpu.getEvent(&a, &g, &temp);

  float acX = abs(a.acceleration.x);
  float acY = abs(a.acceleration.y);
  float acZ = abs(a.acceleration.z - 9.8); // remover gravidade

  float acTotal = max({acX, acY, acZ});

  if (acTotal > LIMIAR_IMPACTO) {
    contagemImpactos++;
    Serial.printf("💥 IMPACTO #%d detectado! Intensidade: %.1f m/s²\n",
                   contagemImpactos, acTotal);
  }

  // Log contínuo a cada 5s
  static unsigned long ultimo = 0;
  if (millis() - ultimo > 5000) {
    Serial.printf("Impactos registrados: %d | Última aceleração: %.1f m/s²\n",
                   contagemImpactos, acTotal);
    ultimo = millis();
  }

  delay(100); // 10Hz de amostragem
}

PROJETO 08 · ANTI-VIOLAÇÃO
Lacre de Luz — Sensor LDR (Anti-Violação)
Detecta abertura indevida da embalagem retornável sem usar lacres plásticos descartáveis
+90 XP
▼
▸ CONTEXTO DO PROBLEMA

Embalagens de alimentos e remédios precisam garantir que não foram abertas antes da entrega. Hoje isso é feito com lacres plásticos descartáveis — mais lixo. A solução elegante: um LDR (sensor de luz) instalado dentro da caixa. Se a caixa for aberta, entra luz e o ESP32 registra a violação — sem gerar nenhum resíduo plástico.

1
Hardware Necessário
COMPONENTEPINO ESP32FUNÇÃO
LDR (fotoresistor)GPIO 34 (ADC)Detecta luz dentro da caixa

2
Circuito Divisor de Tensão
▸ DIVISOR DE TENSÃO COM LDR
3.3V──→LDR──→GPIO 34──→R 10kΩ──→GND
Monte o circuito: 3.3V → LDR → nó → R10k → GND. O GPIO 34 lê o nó do meio. Com mais luz, o LDR tem menos resistência e a tensão no nó aumenta → valor analógico maior. Caixa fechada = escuro = valor baixo. Caixa aberta = luz = valor alto.
3
Código do Projeto
C++ / ARDUINO
// Projeto 8 — Lacre de Luz (Anti-Violação LDR)
const int LDR_PIN  = 34;
const int LIMIAR   = 800; // ajuste: acima = luz detectada

bool violacaoRegistrada = false;
unsigned long horaViolacao = 0;

void setup() {
  Serial.begin(115200);
  Serial.println("Lacre de Luz ativo. Embalagem fechada e monitorada.");
  Serial.printf("Leitura inicial: %d\n", analogRead(LDR_PIN));
}

void loop() {
  int leitura = analogRead(LDR_PIN);

  if (leitura > LIMIAR && !violacaoRegistrada) {
    violacaoRegistrada = true;
    horaViolacao = millis() / 1000; // segundos desde boot
    Serial.println("🚨 VIOLAÇÃO DETECTADA! Embalagem foi aberta indevidamente.");
    Serial.printf("   Momento da violação: %lus após inicialização\n", horaViolacao);
    Serial.printf("   Intensidade de luz: %d/4095\n", leitura);
  }

  if (!violacaoRegistrada) {
    Serial.printf("✅ OK | Luz: %d | Embalagem íntegra.\n", leitura);
  } else {
    Serial.println("⚠ EMBALAGEM COMPROMETIDA — Registrar ocorrência.");
  }

  delay(500);
}

Calibre o Limiar
Com a caixa fechada, anote o valor do LDR (deve ser baixo, <400). Com a caixa aberta em ambiente iluminado, o valor deve subir para 2000+. Defina o LIMIAR em um ponto intermediário seguro (ex: 800). Se o ambiente de transporte for muito escuro, ajuste para baixo.

Código do Projeto
C++ / ARDUINO
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

// Endereço I2C do LCD: 0x27 (mais comum) ou 0x3F
LiquidCrystal_I2C lcd(0x27, 16, 2);
const int PIR_PIN = 15;

// Passos de instrução — máximo 16 chars por linha
const char* passos[][2] = {
  {"PASSO 1:",        "Retire adesivo"},
  {"PASSO 2:",        "Abra aba lateral"},
  {"PASSO 3:",        "NAO RASGAR abas"},
  {"PASSO 4:",        "Achate a base"},
  {"PASSO 5:",        "Deposite aqui!"},
  {"OBRIGADO!",       "Voce fez a dif."}
};
const int NUM_PASSOS = 6;

void setup() {
  Serial.begin(115200);
  pinMode(PIR_PIN, INPUT);
  lcd.init();
  lcd.noBacklight(); // Backlight desligado em repouso
  lcd.clear();
  Serial.println("Totem Maker ativo. Aguardando presença...");
}

void exibirPassos() {
  lcd.backlight();
  for (int i = 0; i < NUM_PASSOS; i++) {
    lcd.clear();
    lcd.setCursor(0, 0); lcd.print(passos[i][0]);
    lcd.setCursor(0, 1); lcd.print(passos[i][1]);
    Serial.printf("%s %s\n", passos[i][0], passos[i][1]);
    delay(3000); // 3s por passo
  }
  lcd.clear();
  lcd.noBacklight();
}

void loop() {
  int presenca = digitalRead(PIR_PIN);

  if (presenca == HIGH) {
    Serial.println("Pessoa detectada! Exibindo instruções...");
    exibirPassos();
    delay(5000); // Pausa antes de detectar novamente
  }

  delay(200);
}
